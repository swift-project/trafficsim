Version 0.4

03/21/2013 12:48 am

→Added void ConvertPBHToDoubles(unsigned long, double&, double&, double&) to Cvatlib_Network
→Added void ConvertDoublesToPBH(double, double, double, unsigned long&) to Cvatlib_Network

03/21/2013 12:54 pm

→Moved void ConvertPBHToDoubles(unsigned long, double&, double&, double&) to the global namespace
→Moved void ConvertDoublesToPBH(double, double, double, unsigned long&) to the global namespace
→Cvatlib_Network and Cvatlib_Voice_Simple are now implemented as base classes of pure virtual functions
→The constructor and destructor have been removed from Cvatlib_Network and Cvatlib_Voice_Simple
→The delete operator is overridden in Cvatlib_Network to call the Destroy function, which in turn properly calls the destructor of the implementation class. This replaces the destructor
→The delete operator is overridden in Cvatlib_Voice_Simple to call the Destroy function, which in turn properly calls the destructor of the implementation class. This replaces the destructor
→Added Cvatlib_Network* Create_Cvatlib_Network(); as a factory function to create Cvatlib_Network objects. This replaces the constructor
→Added Cvatlib_Voice_Simple* Create_Cvatlib_Voice_Simple(); as a factory function to create Cvatlib_Voice_Simple objects. This replaces the constructor
→Moved some internal/private variables and functions to the implementing classes, they are no longer in the base class of Cvatlib_Network or Cvatlib_Voice_Simple

03/21/2013 01:51 pm

→Added __stdcall to functions

03/24/2013 06:28 pm

→Fixed missing VATLIB_API defines

03/26/2013 08:12 am

→Doxygen documentation completed
→Turns out frequency input is in kilohertz, not standard form. Function documentation updated and info about standard form removed
→All documentation updated to refer to "the user", "the client", or "the program" instead of "you"
→Added new defines NOSTDCALL and DEFNULL for Linux. They disable use of __stdcall and define NULL, respectively
→As part of Doxygen documentation, enums now have the name <enumname>_enum so they don't appear as anonymous
→Removed an unused/not needed ccpPropertyType
→Added information to documentation where char**'s are used mentioning that these string arrays are NULL terminated

03/27/2013 05:30 am

→Updated struct implementations to be listed on the page of the defining class instead of separate pages
→Changelog page now has a table of contents
→typedef int declarations for enum storage now have documentation, allowing usage in functions to contain links to jump to the enums
→Functions are now listed in groups, just like in the header
→Moved @page definitions to a separate file to save space in vatlib.h, especially once new pages detailing usage and examples get added
→Ran spelling check on all documentation

03/27/2013 09:04 pm

→Added exceptions to prevent programmers from doing some things they shouldn't
→Added InvalidNetworkSessionException, InvalidObjectException, InvalidRoomException, NetworkNotConnectedException, NetworkSessionExistsException, ObjectNotSetupException, and RoomNotConnectedException
→Added ExceptionCheck function to Cvatlib_Network & Cvatlib_Voice_Simple
→Most functions in Cvatlib_Network & Cvatlib_Voice_Simple will call ExceptionCheck and pass on any exceptions thrown
→Updated doxygen to include thrown exceptions in method documentation

03/28/2013 06:46 pm

→Changed network connected exception check to always get the underlying network status of the network session
→Added bool IsValid(), bool IsSessionExists(), and bool IsNetworkConnected() to Cvatlib_Network
→Added bool IsValid(), bool IsSetup(), bool IsRoomValid(int), and bool IsRoomConnected(int) to Cvatlib_Voice_Simple
→The inheritance chain for exceptions has changed, all exceptions now derive from the new VatlibException class which derives from std::exception

05/03/2013 04:22 pm

→Made what() function public in exception classes
→Fixed exception always being thrown in Cvatlib_Network::CreateNetworkSession
→Minor 64-bit compatibility fixes
